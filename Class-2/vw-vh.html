What Are Viewport Units?

Viewport units are relative length units based on the size of the viewport (the visible portion of the browser window). They're powerful tools for creating truly responsive designs that adapt to different screen sizes in a more fluid way than traditional pixel or percentage-based approaches.

The four main viewport units are:

  • vw (Viewport Width): 1vw equals 1% of the viewport's width
  • vh (Viewport Height): 1vh equals 1% of the viewport's height
  • vmin: 1vmin equals 1% of the viewport's smaller dimension (height or width)
  • vmax: 1vmax equals 1% of the viewport's larger dimension (height or width)

This tutorial will focus primarily on vw and vh, which are the most commonly used viewport units.

Comparing Viewport Units to Other Units

Unit Description Responsive to Use Cases
px (pixels) Fixed unit, absolute length Nothing (static) Borders, small spacing, fixed-size elements
% (percentage) Relative to parent element Parent container size Width of containers, nested responsive layouts
em Relative to element's font size Element's font size Typography, padding related to text
rem Relative to root element's font size Root font size Consistent spacing across the site
vw 1% of viewport width Viewport width Full-width elements, horizontal spacing
vh 1% of viewport height Viewport height Full-height sections, vertical spacing

Percentage vs. vw

Responsive Typography with vw

Viewport units can be used for responsive typography that scales with the viewport size. This eliminates the need for multiple font-size declarations in media queries.

The paragraph text can also use viewport units to scale proportionally.

Tip: Clamping for Better Control

The clamp() function allows you to set minimum and maximum values along with a preferred viewport-relative value:

font-size: clamp(16px, 3vw, 32px);

This ensures text is never smaller than 16px or larger than 32px, but scales fluidly with the viewport in between those values.

Common Use Cases for Viewport Units

1. Full-height Sections

Create sections that are exactly the height of the viewport:

.hero-section {
    height: 100vh;
    /* This creates a section that is exactly the height of the viewport */
}

2. Responsive Typography

Font sizes that scale with the viewport:

h1 {
    font-size: 5vw; /* Scales with viewport width */
}

/* With fallback and clamping for better control */
h1 {
    font-size: calc(24px + 1vw); /* Combines fixed and responsive sizing */
    font-size: clamp(24px, 5vw, 72px); /* Min: 24px, preferred: 5vw, max: 72px */
}

3. Aspect Ratio Preservation

Maintain aspect ratios for elements:

.video-container {
    width: 80vw;
    height: 45vw; /* 16:9 aspect ratio (9/16 = 0.5625 ≈ 56%) */
}

4. Padding and Margins

Spacing that scales with the viewport:

.section {
    padding: 5vh 5vw; /* Vertical padding (5vh), Horizontal padding (5vw) */
    margin-bottom: 3vh;
}

Tips and Best Practices

When to Use Viewport Units

  • For full-viewport layouts (hero sections, full-screen overlays)
  • When you need elements to scale proportionally with the viewport
  • For maintaining specific aspect ratios
  • For responsive typography that scales with the screen

Limitations and Considerations

  • Mobile browsers and vh issues: Mobile browsers sometimes have issues with vh units due to dynamic toolbars
  • Extreme viewport sizes: At very small or very large viewport sizes, scaling can become problematic
  • Accessibility concerns: Text that scales too small or too large can create readability issues
  • Use clamping: Always consider using min(), max(), or clamp() to set boundaries

Hybrid Approaches

Often the best approach is to combine viewport units with other units:

/* Combining units for more control */
.element {
    /* Responsive but with minimum size */
    width: max(300px, 50vw);
    
    /* Responsive padding with minimum */
    padding: max(20px, 5vh) max(20px, 5vw);
    
    /* Hybrid font sizing */
    font-size: calc(16px + 1vw);
}

Browser Support

Viewport units (vw, vh, vmin, vmax) are supported in all modern browsers. The clamp(), min(), and max() functions have good support in modern browsers but may need fallbacks for older browsers.
