Semantic Elements vs Generic Elements

Semantic vs Generic Elements

HTML elements can be divided into semantic and non-semantic elements:

  • Semantic elements: Clearly describe their meaning/purpose to both browser and developer (e.g., <section>, <article>, <header>)
  • Non-semantic elements: Don't describe their content (e.g., <div>, <span>)

Common Semantic Elements

Here are some of the most useful semantic elements in HTML5:

<header> - Represents introductory content, typically a group of introductory or navigational aids. May contain logo, heading elements, navigation, etc.
<nav> - Represents a section of the page whose purpose is to provide navigation links.
<section> - Represents a standalone section of content. Typically has a heading.
<article> - Represents a self-contained composition that could be distributed and reused independently (e.g., blog post, news article).
<footer> - Represents a footer for its nearest sectioning content or sectioning root element. Typically contains information about the author, copyright, links, etc.
<div> - Generic container with no semantic meaning. Used for styling purposes or as a container. <span> - Generic inline container with no semantic meaning. Used for styling parts of text.

CSS Display Properties

Overview of Display Properties

The CSS display property determines how an element is displayed on the page. It affects both the element's external positioning among other elements (block vs. inline) and how its children are laid out (flex, grid).

Display Value Description Example Elements
block Takes up the full width available, starts on a new line div, p, h1, section, article
inline Takes only as much width as needed, does not break to a new line span, a, strong, em
inline-block Like inline but can have width/height, padding and margins button, input, select (can be set on any element)
flex Container becomes a flex container, children become flex items Set manually on containers
grid Container becomes a grid container, children become grid items Set manually on containers
none Element is completely removed from the layout Used to hide elements

Inline Elements

Inline Elements

Inline elements:

  • Do not start on a new line
  • Only take up as much width as necessary
  • Cannot have width/height set (ignored)
  • Vertical margins are ignored
span { background-color: #ffe8e8; padding: 5px; border: 1px dashed #ff8fa3; margin: 0 5px; /* These have NO effect on inline elements */ width: 200px; /* Ignored! */ height: 100px; /* Ignored! */ margin-top: 20px; /* Ignored! */ margin-bottom: 20px; /* Ignored! */ }

This paragraph contains inline elements that flow with the text and don't break the flow. Notice how they wrap naturally with the text and ignore width/height settings.

Block Elements

Block Elements

Block elements:

  • Always start on a new line
  • Take up the full width available by default
  • Can have width/height set
  • Respect all margin settings
p, div { background-color: #e0f8ff; padding: 10px; border: 1px solid #90e0ef; margin-bottom: 10px; /* These ARE respected for block elements */ width: 80%; /* Works! */ margin-top: 20px; /* Works! */ }

This is a block element (p tag). It takes up the full width available and starts on a new line.

This is another block element. Notice how each one starts on a new line.

Inline-Block Elements

Inline-Block Elements

Inline-block elements:

  • Flow with text like inline elements (don't force new lines)
  • Respect width, height, margins like block elements
  • Perfect for creating grid-like layouts without using flex/grid
.demo-element { display: inline-block; background-color: #f0fff0; width: 150px; height: 100px; padding: 10px; margin: 5px; border: 1px solid #b8f2b8; vertical-align: middle; }

These elements are inline-block which means they flow with text but can also have width and height set on them.

Flexbox Layout

Flexbox Layout

Flexbox is a one-dimensional layout method:

  • Great for laying out items in a row or column
  • Allows flexible item sizes that can grow or shrink
  • Provides easy alignment and distribution of space
  • Perfect for navigation bars, card layouts, centering content
.flex-container { display: flex; background-color: #f8edeb; padding: 10px; border: 1px solid #e6beae; /* Try these properties */ justify-content: space-between; /* Controls horizontal distribution */ align-items: center; /* Controls vertical alignment */ flex-wrap: wrap; /* Controls wrapping behavior */ } .flex-item { background-color: #e6ccb2; padding: 20px; margin: 5px; border: 1px solid #c6ac8f; /* Makes items grow/shrink proportionally */ flex: 1; }

Grid Layout

Grid Layout

Grid is a two-dimensional layout method:

  • Perfect for complex layouts with rows and columns
  • Allows precise placement of items in a grid
  • Can create asymmetrical layouts easily
  • Great for overall page layout and complex components
.grid-container { display: grid; grid-template-columns: 1fr 1fr 1fr; /* Three equal columns */ grid-gap: 10px; /* Gap between grid items */ background-color: #dbe7e4; padding: 10px; border: 1px solid #b4c9c3; } .grid-item { background-color: #b4c9c3; padding: 20px; border: 1px solid #87aca3; text-align: center; } /* Span items across multiple columns/rows */ .grid-item-wide { grid-column: span 2; /* Takes up 2 columns */ }

Comparison of Display Types

When to Use Each Display Type
Display Type Best Use Cases Limitations
block
  • Page sections
  • Paragraphs
  • Headers
  • Dividing content
  • Always creates line breaks
  • Not suitable for horizontal layouts
inline
  • Text formatting
  • Links within text
  • Icons in text
  • Cannot set width/height
  • Vertical margins ignored
  • Limited layout control
inline-block
  • Navigation items
  • Buttons in a row
  • Simple grid layouts
  • Whitespace between elements
  • Limited alignment options
  • No built-in responsive features
flex
  • Navigation menus
  • Card layouts
  • Centering content
  • Equal-height columns
  • Only one-dimensional (row OR column)
  • Complex nesting needed for 2D layouts
grid
  • Page layouts
  • Photo galleries
  • Dashboard layouts
  • Complex component layouts
  • More complex syntax
  • Steeper learning curve
  • Older browser support issues

Summary and Best Practices

When to Use Each Type of Element & Display Property

Semantic Elements

  • Use semantic elements whenever possible for better accessibility and SEO
  • Use <section> for thematic grouping of content
  • Use <article> for self-contained content
  • Use <header> and <footer> for introductory and closing content
  • Use <nav> for navigation links

Display Properties

  • Use block for full-width elements that need to be stacked vertically
  • Use inline for text-level elements that should flow with text
  • Use inline-block when you need elements in a row but with set dimensions
  • Use flex for one-dimensional layouts (either row or column)
  • Use grid for two-dimensional layouts (rows and columns together)

Modern CSS Layout Tips

  • Prefer flexbox and grid over older techniques like floats
  • Use semantic HTML elements first, then apply display properties as needed
  • Build responsive layouts using a combination of flex, grid, and media queries
  • For simple components (navigation, card lists), flexbox is often enough
  • For complex page layouts, grid provides more control
  • Remember that flexbox and grid can be nested inside each other
